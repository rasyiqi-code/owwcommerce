# Contribution Guidelines

Thank you for your interest in contributing to OwwCommerce! We appreciate every bit of help, from bug reporting to major feature additions.

## Getting Started

1. **Fork** this repository.
2. **Clone** your fork to your local environment.
3. Create a new **Branch** for your feature or fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. Make your changes and ensure your code follows the standards.

## Coding Standards

We follow the **PSR-12** standard for PHP and use modern Vanilla JavaScript (ES6+). Some important rules:
- Use 4-space indentation for PHP and 2-space indentation for JS.
- Provide clear comments for complex functions or logic.
- Ensure no hardcoded data in the core logic.

## Pull Request Workflow

1. Ensure your code is tested locally.
2. Push your branch to your forked repository.
3. Open a **Pull Request** to the `main` branch of the original repository.
4. Clearly describe your changes in the PR description.

## Reporting Bugs

Please use the GitHub **Issues** feature to report bugs. Include:
- Steps to reproduce the bug.
- Expected vs actual result.
- PHP and WordPress versions used.

Thank you for helping make OwwCommerce better! 🚀
